# 動漫資訊站資料欄位設計

## 動畫資料 anime

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | number | 動畫編號 |
| title | string | 動畫名稱 |
| image | string | 海報圖片網址 |
| description | string | 動畫介紹 |
| genre | string | 類型，例如奇幻、校園、冒險 |
| season | string | 播出季度，例如 2026 春 |
| episodes | number | 集數 |
| status | string | 狀態，例如連載中、已完結 |
| platformId | number | 對應平台編號 |

### 範例
```json
{
  "id": 1,
  "title": "鬼滅之刃",
  "image": "/images/demon-slayer.jpg",
  "description": "講述少年與鬼戰鬥的故事。",
  "genre": "奇幻",
  "season": "2026 春",
  "episodes": 12,
  "status": "連載中",
  "platformId": 1
}
```

## 平台資料 platform

| 欄位 | 型別 | 說明 |
| --- | --- | --- |
| id | number | 平台編號 |
| name | string | 平台名稱 |
| website | string | 平台網址 |
| price | string | 基本方案價格 |
| description | string | 平台介紹 |

### 範例
```json
{
  "id": 1,
  "name": "Netflix",
  "website": "https://www.netflix.com",
  "price": "290元 / 月",
  "description": "提供多種類型動畫與影集。"
}
```

## 第一版建議先做的資料筆數
- 平台：3 到 5 個
- 動畫：10 到 20 部

## 第一版建議平台
- Netflix
- 巴哈姆特動畫瘋
- Crunchyroll
- Disney+
- friDay 影音
